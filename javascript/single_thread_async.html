Demo Single-Threaded vs Asíncrono

¿Por qué importa el modelo single-threaded?

JavaScript ejecuta todo el código en un solo hilo. Si una tarea es muy pesada o bloqueante, detiene la ejecución de todo lo demás, incluyendo la interfaz de usuario. Por eso, para manejar múltiples tareas de entrada/salida (I/O) y computación pesada, se usan técnicas asíncronas y entornos como Node.js o Deno que permiten gestionar muchas operaciones concurrentes sin bloquear el hilo principal.

  • Operación Bloqueante: El navegador se congela hasta que termina la tarea.
  • Operación Asíncrona: El navegador sigue funcionando y la tarea se completa en segundo plano.

En esta demo puedes ver cómo una operación bloqueante detiene la página, mientras que una operación asíncrona permite que el hilo principal siga libre y respondiendo. Esto es fundamental para aplicaciones web modernas y servidores eficientes.
